# LocaL-Kino

Домашний кинотеатр: каталог фильмов с rutracker.net и локальный стриминг через libtorrent.

Проект собирает несколько коллекций rutracker, сохраняет данные в `data/`, генерирует каталог `index-kino.html` и запускает локальный Flask-сервер для просмотра.

## Быстрый старт

```powershell
.venv\Scripts\Activate.ps1
python generate_page.py --refresh --quick
python stream_server.py
```

Сервер открывается на порту из `.env` (`SERVER_PORT`). В текущем проекте используется `14876`:

```text
http://localhost:14876
```

## Основные команды

| Команда | Назначение |
|---------|------------|
| `python generate_page.py` | Сгенерировать HTML из кеша |
| `python generate_page.py --refresh` | Обновить все коллекции |
| `python generate_page.py --refresh --fast` | Быстро обновить каталог без тяжёлого обогащения |
| `python generate_page.py --refresh --quick` | Быстро обновить первую страницу |
| `python generate_page.py --refresh --collection=kino_sng` | Обновить одну коллекцию |
| `python generate_page.py --refresh --collection=kino_sng --limit 10` | Взять до 10 новых тем из коллекции |
| `python generate_page.py --collection=kino_sng` | Сгенерировать HTML по кешу одной коллекции |
| `python stream_server.py` | Запустить веб-сервер |

## Коллекции

Коллекции описаны в `generate_page.py` в словаре `COLLECTIONS`.

| ID | Название | Раздел rutracker | Возрастная чистка |
|----|----------|------------------|-------------------|
| `nashe_kino` | Наше кино | `f=22` | Да |
| `kino_sng` | Фильмы ближнего зарубежья | `f=2540` | Нет |
| `novinki_2026` | Новинки 2026 | `f=252` | Да |
| `kino_sng_hd` | Фильмы Ближнего Зарубежья (HD Video) | `f=1247` | Нет |

`MAX_TOPICS=30` - это лимит пополнения за один refresh, а не жесткий максимум коллекции. Если в коллекции уже больше 30 валидных тем и возрастная чистка их не удалила, такие темы остаются.

Темы без magnet не считаются полезными для каталога и не должны попадать в итоговую коллекцию.

## Автообновление

Сервер умеет запускать refresh автоматически:

- при старте после housekeeping;
- затем периодически, примерно раз в 12 часов;
- refresh выполняется только если дневная метка `data/last_refresh_date.txt` отсутствует или отличается от текущей даты.

Метка содержит только дату без времени. После успешного refresh сервер обновляет эту метку.

Чтобы веб-сервис продолжал работать во время обновления, refresh использует staging-каталог `data/staging_refresh`, а затем переносит готовые данные в основную папку.

Автообновление сервера запускает быстрый refresh (`--fast`): новые темы добавляются с magnet/poster, а рейтинги, трейлеры и дополнительные постеры можно догнать полным ручным refresh.

Параллельные сетевые операции используют общий параметр `.env` `WORKER_COUNT` (в проекте `8`): он применяется к загрузке magnet/poster и фоновому enrich.

Кеш страниц форума (`data/topic_cache/f*_p*.html`) используется только как fallback при ошибке сети. При refresh сервис сначала пытается скачать свежий listing с rutracker. Максимальный возраст fallback-кеша задаёт `.env` параметр `LISTING_CACHE_MAX_AGE_DAYS`, по умолчанию `1`.

## Housekeeping

`TOPIC_MAX_AGE_DAYS` в `.env` задает возраст темы в днях, по умолчанию `90`.

Возрастная чистка применяется не ко всем коллекциям, а только к коллекциям с `age_cleanup=True`. Это нужно, чтобы агрессивные коллекции очищались, а редкие и малоизменяемые коллекции не теряли полезные старые темы.

Старые темы удаляются только если коллекция остается больше `MAX_TOPICS`. Удаленные темы архивируются в `data/topics_archive.json`. Вместе с удаленными темами чистятся связанные локальные данные: topic-cache и неиспользуемые постеры.

Если у темы неизвестна дата регистрации в системе, она должна обогащаться текущей датой, чтобы housekeeping мог работать предсказуемо.

## Постеры

Постеры сохраняются в `data/posters/`. Если постер из темы недоступен или указывает на проблемный хостинг, используется fallback-поиск постера. Для заглушки используется локальный файл:

```text
data/posters/placeholder.png
```

## Просмотр и browse-режимы

Основной каталог открывается через сервер. Дополнительно в `stream_server.py` есть browse-страницы:

- `/test` - каталог;
- `/browse/carousel` - карусель;
- `/browse/random` - случайный фильм;
- `/browse/filter` - фильтр;
- `/browse/timeline` - хронология;
- `/browse/shuffle` - режим ТВ.

На карточках цвет кнопки просмотра зависит от типа контейнера: для файлов, которые требуют альтернативного пути live-просмотра, кнопка выделяется иначе. Сообщения пользователю при этом не должны технически перечислять контейнеры, а должны говорить простым языком, что найден быстрый способ live-просмотра.

## Важные файлы

- `generate_page.py` - парсинг rutracker, обогащение данных, генерация `index-kino.html`.
- `stream_server.py` - Flask-сервер, каталог, browse-страницы, авто-refresh, housekeeping, стриминг.
- `engine.py` - libtorrent-движок, выбор видеофайла, приоритеты кусков, readahead.
- `player.html` - страница плеера.
- `config.py` - чтение `.env`.
- `project_io.py` - пути проекта и атомарная запись JSON.
- `.env.example` - шаблон настроек.

## Публикация

Для публикации наружу включается `PUBLIC_MODE=1` в `.env`. В этом режиме сервер не раздаёт корень проекта как static, наружу доступны только явно разрешённые файлы (`/`, `/player.html`, `/data/posters/*`) и browse-страницы. Админские ручки `/refresh`, `/cleanup`, `/hide`, `/unhide`, `/enrich/*` отключены, а `/watch` принимает только magnet из текущего каталога.

## Рабочие данные

Эти файлы и папки генерируются локально и не попадают в git:

- `data/temp/` - временные скачанные видео.
- `data/posters/` - постеры и заглушка.
- `data/topic_cache/` - кеш страниц rutracker.
- `data/imdb/` - локальные IMDB справочники `title.ratings.tsv.gz` и `title.basics.tsv.gz`; если ID не найден в старом файле, скачивается свежий файл и становится основным.
- `data/staging_refresh/` - временные данные refresh.
- `data/torrents_data.json` - основной каталог.
- `data/topics_archive.json` - архив удаленных старых тем.
- `data/last_refresh_date.txt` - дата последнего успешного авто-refresh.
- `data/index-kino.html` - сгенерированная HTML-страница.
- `data/*_cache.json` - вспомогательные кеши.

Перед запуском сервера проверь размер `data/temp/`: при превышении `MAX_TEMP_SIZE_GB` сервер может удалить старые файлы из этой папки.
