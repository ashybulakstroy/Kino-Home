# Оптимизация времени refresh

Текущий pipeline полностью последовательный (кроме шага 3).

## Где теряется время

### 1. Парсинг коллекций (шаг 1) — последовательный
Сейчас коллекции обрабатываются по одной. Если их 5, каждая ждёт завершения предыдущей.
**Фикс**: `ThreadPoolExecutor` по коллекциям. Каждая коллекция парсится в своём потоке.
**Риск**: блокировка записи в `TORRENTS_CACHE` при слиянии — нужна аккуратная синхронизация или merging после завершения всех потоков.

### 2. Загрузка IMDB со страниц world-источников (шаг 3b) — последовательный цикл
`fetch_world_imdb_ids(world_new)` — цикл по темам, каждая HTTP-запрос.
**Фикс**: обернуть в `ThreadPoolExecutor(WORKER_COUNT)`.

### 3. Поиск IMDB ID (шаг 4) — последовательный цикл
`search_imdb_ids(all_new_topics)` — цикл с HTTP-запросами.
**Фикс**: `ThreadPoolExecutor(WORKER_COUNT)`.

### 4. Поиск Кинопоиск ID (шаг 5) — последовательный цикл
`search_kinopoisk_ids(all_new_topics)` — цикл с HTTP-запросами.
**Фикс**: `ThreadPoolExecutor(WORKER_COUNT)`.

### 5. Обогащение данных (шаг 8) — последовательный цикл
`enrich(all_new_topics, ratings, basics)` — внутри цикл по темам, каждая `enrich_topic()`.
**Фикс**: `ThreadPoolExecutor(WORKER_COUNT)`.

### 6. Кинопоиск постеры (шаг 10) — последовательный цикл
Цикл по всем темам, скачивание постеров.
**Фикс**: `ThreadPoolExecutor(WORKER_COUNT)`.

## Дополнительные идеи

### 7. Инкрементальный refresh
Сейчас даже если новых тем нет — проход по всему кешу (fix_bad_topics, repair_world_titles, dedup, генерация HTML).
**Фикс**: если новых тем 0 и не `--force` — пропустить всё, кроме генерации HTML.

### 8. Кеш поиска IMDB
`search_imdb_ids()` не кеширует результаты между refresh-ами (кроме topic-страниц).
**Фикс**: сохранять `imdb_search_cache.json` по названию фильма.

### 9. Скачивание IMDB датасета
Уже есть проверка на 7 дней. Можно сделать опциональный флаг `--skip-imdb-dataset` для ultra-fast refresh.
